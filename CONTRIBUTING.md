# Contributing to CollaborNest

Thank you for your interest in contributing to CollaborNest! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub. When reporting issues, please include:

- A clear description of the problem
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Your environment (OS, Node.js version, etc.)

### Pull Requests

1. Fork the repository
2. Create a new branch for your feature or bugfix
3. Make your changes
4. Add tests for your changes
5. Run `npm test` to ensure all tests pass
6. Run `npm run lint` to ensure code style compliance
7. Run `npm run build` to ensure the code builds successfully
8. Commit your changes with a clear commit message
9. Push your branch to your fork
10. Open a pull request

### Code Style

We use ESLint and Prettier to maintain code quality and consistency. Please ensure your code passes all linting checks before submitting a pull request.

### Testing

Please add tests for any new features or bug fixes. We use Jest for testing.

### Documentation

If you add new features, please update the documentation in the README.md file.

## Development Setup

1. Clone the repository
2. Install dependencies: `npm install`
3. Make your changes
4. Build the project: `npm run build`
5. Run tests: `npm test`
6. Run linter: `npm run lint`

## Code of Conduct

Please be respectful and considerate of others when contributing to this project.

## Questions?

If you have any questions, feel free to open an issue or start a discussion.

Thank you for contributing to CollaborNest!
